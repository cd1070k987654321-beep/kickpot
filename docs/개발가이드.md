# ⚽ KickPot 모바일 MVP 개발 가이드 v2.0

---

## 1. 제품 방향

KickPot은 초기 버전을 **모바일 전용 MVP**로 개발한다.
목표는 데스크톱 대응이 아니라, 스마트폰에서 팀 운영 핵심 흐름을 빠르게 검증하는 것이다.

### 핵심 목표
- 팀 생성 / 가입
- 매칭 생성 / 요청 / 수락
- 참여자 투표
- 경기 기록 작성 / 승인

## 3. 프론트 구조 원칙

### 모바일 전용 UI 원칙
- 390~430px 폭 기준으로 설계
- 하단 탭바 중심
- 풀스크린 단일 컬럼 우선
- 데스크톱 확장은 추후 대응

### 주요 라우트
- `/`
- `/login`
- `/signup`
- `/home`
- `/team`
- `/team/[teamId]`
- `/match`
- `/match/[matchId]`
- `/profile`

## 5. DB 핵심 엔티티

### User / Profile
- id
- email
- nickname
- profile_image
- bio
- main_position
- preferred_positions

### Team
- id
- name
- logo_url
- cover_image_url
- description

### TeamMember
- id
- user_id
- team_id
- role (owner / member)

### Match
- id
- team_a_id
- team_b_id
- date
- location
- status (pending / requested / accepted / rejected / completed)

### Vote
- id
- team_id
- match_id
- type (date / participant)
- status (open / closed)

### Lineup
- id
- match_id
- user_id
- position

### Record
- id
- match_id
- score_a
- score_b
- status (draft / reviewing / rejected / approved)
