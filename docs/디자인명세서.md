# 🎨 KickPot (킥팟) 모바일 디자인 명세서 v2.1

---

## 1. 디자인 방향

KickPot은 **모바일 전용 MVP**로 설계한다.
초기 목표는 반응형 확장이 아니라, **스마트폰에서 가장 빠르게 매칭을 찾고 팀 운영 액션을 처리하는 UI**다.

### 키워드
- Dark
- Neon
- Sporty
- Fast
- Thumb-friendly
- Mobile-first

### 한 줄 정의
“한 손으로 매칭을 찾고 팀 운영을 끝내는 모바일 축구 앱”

---

## 2. 기본 디바이스 기준

### 기준 뷰포트
- 390px ~ 430px 폭 우선
- iPhone / Galaxy 일반 모델 기준

### 안전 원칙
- 모든 핵심 액션은 한 손 조작 가능해야 함
- 주요 CTA는 하단 영역에 배치
- 상단보다 하단 탭/바텀시트 활용 우선

## 6. 핵심 컴포넌트

### 버튼
- 기본 높이: 52px
- Primary: 네온 라임 배경 + 어두운 텍스트
- Secondary: 다크 배경 + 라임 보더
- 하단 고정 CTA 허용

### 카드
- 배경: #111827
- Radius: 20px
- 매칭 핵심 정보가 첫 화면에 보여야 함
  - 날짜
  - 시간
  - 지역
  - 구장
  - 팀 수준
  - 상태

### 필터 칩
- 지역
- 날짜
- 레벨
- 구장 형태

### 하단 탭바
- 홈 / 팀 / 매치 / 프로필
- 현재 탭 명확한 강조

---

## 7. 주요 화면 구조

### 7-1. 랜딩
- 서비스 한 줄 소개
- 로그인 / 회원가입 CTA
- MVP 핵심 기능 요약

### 7-2. 홈 = 매칭 탐색 메인
- 상단 검색 의도 문구
- 지역/날짜/레벨 필터 칩
- 추천 매칭 카드 리스트
- 카드에서 바로 “요청 보내기” 가능
- 하단 CTA: “내 매치 만들기”

### 7-3. 팀
- 팀 요약 헤더
- 멤버 / 게시판 / 매치 탭
- 팀 생성 / 초대 버튼은 모바일 하단 액션 우선

### 7-4. 매치 상세
- 경기 정보 카드
- 상태 카드
- 참여자 / 라인업 / 기록 섹션
- 상태에 따른 CTA 노출

### 7-5. 프로필
- 닉네임 / 포지션 / 소개
- 최근 경기 기록 요약

### 7-6. 인증 화면
- 단일 컬럼
- 입력 필드 3개 이하 유지
- 버튼은 항상 풀너비
- 이메일 인증 안내는 카드형 메시지로 분리

---

## 8. 인터랙션 원칙

- 클릭 시 살짝 축소
- 로딩은 skeleton 또는 버튼 상태 변경
- 모달보다 bottom sheet 우선 고려
- 중요한 액션은 하단 고정 버튼 우선
- 홈에서는 스크롤 중에도 주요 CTA가 사라지지 않게 설계

---

## 9. UX 핵심 원칙

- 홈에 들어오자마자 바로 매칭을 찾을 수 있어야 함
- 상태가 곧 액션
- 필터 → 카드 확인 → 요청 보내기 흐름이 짧아야 함
- 텍스트보다 카드 구조로 빠르게 이해
