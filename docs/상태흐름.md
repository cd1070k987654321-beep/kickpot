# ⚽ KickPot 상태 흐름 (State Machine) v1.0

---

## 🔥 1. Match (매칭 상태)

### 상태 정의
- pending (요청됨)
- requested (상대팀에 요청 보냄)
- accepted (수락됨)
- rejected (거절됨)
- completed (경기 종료)

---

### 상태 흐름

pending
→ requested (매칭 요청 보냄)
→ accepted (상대팀 수락)
→ completed (경기 진행 완료)

requested
→ accepted
→ rejected

accepted
→ completed

---

### 액션 & 권한

- 매칭 생성 → Team A
- 요청 보내기 → Team A
- 수락 / 거절 → Team B
- 완료 처리 → 양 팀

---

### UX 규칙

- pending: 요청 버튼 표시
- requested: “대기중” 표시
- accepted: 라인업 / 참여자 기능 활성화
- completed: 기록 입력 가능

---

## 👥 2. Participant (참여자 확정 상태)

### 상태 개념
- 투표 기반 → 확정 리스트 생성

---

### 흐름

투표 생성
→ 투표 참여
→ 투표 종료
→ 참여자 확정

---

### 규칙

- 투표 종료 전까지 변경 가능
- 확정 후 변경 제한

---

## 🧠 3. Lineup (라인업 상태)

### 상태
- not_set (미설정)
- configured (설정됨)

---

### 흐름

not_set
→ configured (라인업 저장)

---

### 규칙

- accepted 상태에서만 가능
- 팀장 또는 권한자만 수정 가능

---

## 📊 4. Record (경기 기록 상태) ⭐ 핵심

### 상태 정의

- draft (작성됨)
- reviewing (상대팀 확인중)
- rejected (거절됨)
- approved (확정됨)

---

### 상태 흐름

draft
→ reviewing (확인 요청)

reviewing
→ approved (상대 승인)
→ rejected (상대 거절)

rejected
→ draft (수정 후 재요청)

approved
→ 종료 (변경 불가)

---

### 규칙

- 승인 전까지 수정 가능
- 승인 후 수정 불가
- 양 팀 중 한쪽이라도 거절 시 확정 불가

---

### UX 규칙

- reviewing: 승인/거절 버튼 표시
- rejected: 수정 안내 표시
- approved: 읽기 전용

---

## 🗳️ 5. Vote (투표 상태)

### 상태

- open (진행중)
- closed (종료됨)

---

### 흐름

open
→ closed

---

### 규칙

- open 상태만 참여 가능
- closed 상태는 읽기만 가능

---

## 🔐 6. 권한 흐름 (Role 기반)

### 역할

- Owner (팀장)
- Member (팀원)
- Opponent (상대팀)

---

### 권한 요약

| 기능 | Owner | Member | Opponent |
|------|------|--------|----------|
| 팀 설정 | O | X | X |
| 게시글 작성 | O | O | X |
| 매칭 생성 | O | O | X |
| 매칭 수락 | X | X | O |
| 라인업 설정 | O | (옵션) | X |
| 기록 입력 | O | O | O |
| 기록 승인 | X | X | O |

---

## 🔥 7. 전체 흐름 (핵심)

매칭 생성
→ 매칭 요청
→ 상대팀 수락
→ 투표 (참여자)
→ 참여자 확정
→ 라인업 구성
→ 경기 진행
→ 기록 입력
→ 상대팀 승인
→ 기록 확정
